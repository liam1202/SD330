Liam's Design Principles

Key Principles

  • Ensure Visbility
  • Maintain Consistency
  • Provide Feedback Immediately
  • Apply Constraints
  • Offer Affordances
  • Enhance Accessibility

Complexity Principles

  • Cannot reduce a certain amount of complexity for any system.
  • Necessitate complexity when it comes to apps that have a lot of functionality
  • Make an app have a limit to how simple it can be
  • Allow users to learn the system by doing
  • Reduce clutter without losing functionality
  • Help users adopt more efficient methods

Creativity Principles

  • Do not stick with the first idea, allow yourself to try out others
  • Explore questions and ideas by interacting with suppliers, users, and team mates

Cultural Principles

  • Use phrases, icons, and colors; but make sure they mean the same thing to the users as they do to you
  • Remember that individualist cultures require fewer instructions
  • Concentrate instructions on popular preferences in community cultures
  • Communicate with team mates from different cultures/backgrounds to gain a better understanding

Gender Principles

  • Use gender-neutral language and avoid making assumptions about the gender of users
  • Offer options for users to specify their preferred pronouns and titles
  • Avoid using images or text that reinforce stereotypes or biases related to gender
  • Train system designers and admins to be aware of gender-related issues and to handle them sensitively and effectively.

Conceptional Principles

  • Consider that most users prefer that sites work the same way as other sites they already know
  • When making changes, minimize discord by empowering users to continue using a familiar version for a limited time.
  • Design the system to handle unexpected situations and errors
  • Ensure that the systme is reliable with minimal downtime and consistent performance
  • Concentrate on user-centered designs
  • Ensure the system initiates the interaction; have the user decide whether to respond.

Mind Maps

  • Use a mind map to brainstorm thoughts organically without worrying about order and structure

Bubble Outlines

  • Use a bubble outline to visualize components and processes in the system

Flowcharts

  • Use a flowchart to understand how a process is done

Cognition Design Principles

  • Maintain consistency in terminology and design elements
  • Organize information in a logical hierarchy
  • Present information in digestible chunks
  • Consider the user's limited attention

Human Center Design Principles

  • Create prototypes of the system or its components to test with users
  • Actively listen to user feedback and integrate it into design iterations
  • Establish design goals that align with the needs and goals of the users
  • Create mechanisms for ongoing communication and feedback between users and the design team

Interface Principles

  • Maintain a consistent layout, visual elements, and navigation
  • Offer affordances to users
  • Create the interface to be inclusive
  • Use constraints to apply limits to the final product

Emotional

  • Use stories and anecdotes
  • Use emotional triggers to get user to take an action
  • Make something new or novel to surprise the users
  • Remember that 'look and feel' is the first indicator of trust

Accessibility Principles

  • Make content and interface elements easily perceivable to all users, including those with visual or hearing impairments
  • Design the system so that it can be operated effectively by users with various disabilities
  • Create a user interface and content that is easy to understand
  • Ensure that features and functions can be acessed by a keyboard
  • Include captions for audio and video content
  • Provide alternative text descriptions for images and multimedia to assist users with visual impairments.

Error Principles

  • Provide informative feedback to users when errors occur
  • Maintain consistency in how errors are presented and handled across the system
  • Remember the difference between Mistakes and Slips
